### 정규화

■진행1. 정규화 1NF ~ 5NF

STEP1) 정규화?
1. 관계형 데이터 모델 - 데이터간 관계에 초점을 둔 모델
2. 이상 (Anomaly) - 공간낭비
  - 삽입 , 갱신 , 삭제 이상
3. 정규화? 이상을 제거

STEP2) ERD 핵심구성요소
1.개체(Entity) - 테이블, 개념의 정보단위
2.속성(Attrubute) - 테이블의 열 , 가장 작은 논리전 단위
3.관계(Relationship) - 외래키

■진행2. 정규화 1NF ~ 5NF

STEP1) 1NF - 관계 (최소형)
: 최소한의 테이블, 중복되는 항목이 없어야한다.

STEP2) 2NF - 종속관계
:~ 를 알면 ~ 를 알 수 있다 
 X(결정자) → Y(종속자)
    1. {STD_ID, 과목}               성적
    2. {STD_ID}                    학부
    3. {STD_ID}                    PROFESSOR
    4. {PROFESSOR}                 학부

STEP3.5) BCNF
1. 관계형데이터베이스의 X → Y 성립할때
   모든 결정자가 후보키이면 BCNF가 된다.
2. 식별자로 쓰이는 속성이 일반속성에 종속되지 않아야 한다.
3. 3NF를 통해서도 제거되지 않은 3.5 정규화
※ KEY?
: 튜플(행,레코드)들을 구분할수 있는 기준필드
NO(PK) NAME    AGE   JUMIN
1      장동건   54    720514
2      장동건   54    720514

1) 후보키 - 기본키로 사용할 수 있는 속성(NO,JUMIN)
   - 유일성(유일하게 식별), 최소성(식별시 최소한의 속성)
2) 기본키 - 후보키에서 선택한 주키(main key) - NO 
3) 대체키 - 후보키에서 선택 안된 키 - JUMIN
4) 외래키 - 테이블과 테이블을 연결해주는 속성(deptno)
5) 슈퍼키 - 속성+속성 집한된 구성키
  std_id + course_id
  - 유일성o(다른 튜플 구분o), 최소성(x)

※ 결정자      종속자   
{STD_NAME  ,  COURSE}  →  INSTRUCTOR
{COURSE}  →  INSTRUCTOR

※ 이상확인
insert : 새로운 강사와 과목을 개설하려고 해도, 수강하는 학생이 없으면 data 삽입 불가
update : JAVA 과목의 강사를 SALLY에서 TOM으로 변경, 일부분만 수정시 데이터 불일치
delete : JSP 과목삭제시 학생정보까지 다 삭제

STEP4) 4NF
1. 같은 테이블 내의 독립적인 두개 이상의 컬럼이 다른컬럼에 종속
2. 단일 A와 다중값 B
담당교수 A1, A2
교재코드 Z1, Z2

STEP5) 5NF
1. 조인종속
- 하나의 릴레이션을(테이블) 여러개의 릴레이션으로 분해했다가 다시 조인했을때 데이터 손실없고 필요없는 데이터가 생기는 것을 말함.

◼1. ERD (ENTITY  RELATIONSHIP DIAGRAM)
: 데이터 관계간의 초점에 둔 모델

1. 개체(ENTITY) - 테이블
2. 속성(Attribute) - 열,컬럼
3. 관계(Relationship) - 외래키

◼2. ~ 구성되어 있다
ex1) 하나의 A는 하나의 B로 구성되어있다.
[홍길동] ┼───┼ [주민증]

ex2) 하나의 A는 여러개의 B로 구성되어있다.
:한명의 학생은 여러개의 수강내역을 가진다.
[홍길동] ┼───∈ [JAVA , JSP , SPRING , MYSQL] 

◼3. 점선 VS 실선
- 실선 : 부모테이블의 기본키를 자식테이블의 기본키로 사용
- 점선 : 부모테이블의 기본키를 자식테이블의 기본키로 사용 안 한 경우

■4. FOREIGN KEY
    => 외래키(참조키)
    => 다른테이블의 기본키를 참조하는 키
    => 중복가능 / NULL 허용함
    => 참조되고있는테이블의 데이터 값 이외의 값은 삽입할수 없음.
    => insert할때 잘못된 데이터 삽이 안되도록하는 것
    => 레코드 삭제나 테이블삭제를 할때는 반드시 FOREIGN KEY가
       지정된 레코드나 테이블을 삭제한후에 참조대상을 삭제할수 있다.


    방법
       [ CONSTRAINT 별칭 ] REFERENCES 테이블이름(필드명)
