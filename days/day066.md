2. 스케쥴부품
@Component
public class ApiScheduledTask { 
   @Scheduled(fixedRate = 2000){}   2초마다 실행 
   @Scheduled(fixedDelay = )  어떤작업이 끝난후에 지정된 시간에 시행
   @Scheduled(cron="0 0 10 * * ?")
   public void runTesk2() {
      System.out.println(".......스케쥴러 실행중 : " + System.currentTimeMillis());
   }
   cron="0 0  0  * * ?"    // 0초  0분  0시   일  월 요일- 매일자정
   cron="0 0  12 * * ?"    // 0초  0분  12시  일  월 요일- 매일정오
   cron="0 30 18 * * ?"    // 매일 오후 6시 30분
   cron="0 0  0  1 * ?"    // 매월 1일 자정
   cron="0 0  0  ? * MON"  // 매주 월요일 자정
   
   * 제한없는 모든값
   ? 특정값 없음
   SUN 일, MON 월, TUE 화, WED 수, THU 목, FRI 금, SAT토


>기능 1: AI 기반 프로젝트 리스크 예측 & 자동 알림 항목내용핵심 아이디어진행률·지연 태스크·일정 데이터를 AI로 분석해 지연 가능성 사전 예측, 위험도 높으면 Slack/Discord 알림사용 APIOpenAI API (리스크 점수 산출), Slack/Discord Webhook (실시간 알림)차별 포인트알림에 대안 액션 제안(인력 재배치, 일정 조정) 포함 → 단순 경고가 아닌 의사결정 지원어필 포인트데이터 분석 + 외부 협업 툴 연동

>기능 2: 태스크 자동 요약 & 주간 보고서 생성
항목내용핵심 아이디어태스크 완료/진행 상황을 AI가 요약해 자동 주간 보고서 생성사용 APIOpenAI API (자연어 요약), Google Docs API (문서화 및 공유)차별 포인트 ①정량적 지표 분석 — 텍스트 요약뿐 아니라 완료율·평균 소요시간·일정 초과 빈도를 SQL/백엔드로 계산해 AI 프롬프트에 주입(Context Injection)차별 포인트 ②다중 플랫폼 연동 — Google Docs API로 회사 공식 템플릿에 맞춰 내용 바인딩, .docx/PDF 다운로드 또는 메일 발송차별 포인트 ③역할별 맞춤 보고서 분기 — 팀장용(거시적 리스크·진행률) / 개발자용(기술 이슈·다음 주 작업)으로 프롬프트 분기어필 포인트백엔드 데이터 처리 → AI 요약 → 외부 문서 API 연동까지 풀스택 역량
